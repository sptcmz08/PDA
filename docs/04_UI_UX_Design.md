# 04 UI UX Design

## 1. UX Goal

แอปนี้ต้องออกแบบให้พนักงานใช้งานได้ทันทีโดยแทบไม่ต้องอบรม

หลักการ

- เปิดแอปแล้วพร้อมสแกน
- ปุ่มใหญ่
- ข้อความสั้น
- สีชัด
- เสียงชัด
- ไม่ต้องกดยืนยันหลังยิง
- ไม่ต้องเข้าเมนูซับซ้อน

## 2. Main Screen

แอปมีหน้าหลักเป็น Scanner Screen

ควรเปิดแอปแล้วเข้าหน้านี้ทันที

Layout ตัวอย่าง

```text
┌──────────────────────────────┐
│        สแกนเลขพัสดุ          │
├──────────────────────────────┤

│      จำนวนที่สแกนแล้ว        │
│          125 รายการ          │

│ [ Scanner Input Field ]      │
├──────────────────────────────┤
│                              │
│        สถานะล่าสุด           │

```

## 3. Status Colors

| State | Color | Meaning |
|---|---|---|
| Ready | White/Grey | พร้อมสแกน |
| Success | Green | บันทึกสำเร็จ |
| Duplicate | Red | เลขซ้ำ |
| Invalid | Orange | รูปแบบผิด |
| Error | Red/Dark | Error ระบบ |

## 4. Status Messages

### Ready

```text
พร้อมสแกน
```

### Success

```text
บันทึกสำเร็จ
829362431516
```

### Duplicate

```text
เลขนี้ถูกสแกนแล้ว
829362431516

```

### Invalid

```text
รูปแบบ Tracking ไม่ถูกต้อง
```

## 5. Scanner Input

Requirement

- TextField ต้อง Focus อัตโนมัติ
- หลังสแกนเสร็จต้อง Clear อัตโนมัติ
- หลัง Clear ต้อง Focus กลับทันที
- อาจซ่อน Cursor ได้ถ้าเหมาะสม
- Input Field ควรมีขนาดพอให้ตรวจสอบได้ แต่ไม่ควรเด่นเกินสถานะ

## 6. Button Design

ปุ่มต้องใหญ่ กดง่ายบน PDA

ปุ่มหลัก

- ค้นหา
- ประวัติ
- Export CSV
- เริ่มรอบใหม่

แนวทาง

- ความสูงอย่างน้อย 48–56 px
- ตัวอักษรใหญ่
- ระยะห่างเพียงพอ
- ปุ่มเริ่มรอบใหม่ควรเป็นสีแดง/ส้ม หรือมี Confirm เพื่อป้องกันกดพลาด

## 7. Search Dialog

เมื่อกด "ค้นหา"

แสดง Dialog

```text
ค้นหาเลข Tracking

[กรอกเลข Tracking]

[ค้นหา] [ปิด]
```

ถ้าเจอ

```text
พบข้อมูล
829362431516
วันที่ 18/06/2026
เวลา 14:30:05
```

ถ้าไม่เจอ

```text
ไม่พบข้อมูล
```

## 8. History Screen / Dialog

แสดงรายการล่าสุดก่อน

```text
ประวัติการสแกน

1. 829362431516
   18/06/2026 14:30:05

2. 829360206500
   18/06/2026 14:29:58
```

ควรมี Scroll

## 9. Export Popup

หลัง Export สำเร็จ

```text

[แชร์ไฟล์] [ปิด]
```

กดแชร์ไฟล์แล้วเปิด Android Share Sheet

## 10. Confirm เริ่มรอบใหม่

เมื่อกดเริ่มรอบใหม่ ต้อง Confirm ก่อนเสมอ

```text
ต้องการเริ่มรอบใหม่หรือไม่?

ข้อมูลเดิมจะถูกล้างออกจากเครื่อง

[ยกเลิก] [ยืนยัน]
```

หลังยืนยัน

```text
เริ่มรอบใหม่แล้ว
พร้อมสแกน
```

## 11. Sound UX

ต้องมีเสียงต่างกันชัดเจน

- Success: ติ๊ด 1 ครั้ง
- Duplicate: ติ๊ด ติ๊ด ติ๊ด หรือ Alarm
- Invalid: เสียง Error สั้น ๆ

ถ้าใช้ไฟล์เสียงไม่ได้ ให้ fallback เป็น system beep หรือ vibration หากเครื่องรองรับ

## 12. Accessibility / Usability

- ข้อความภาษาไทยชัดเจน
- ขนาดตัวอักษรใหญ่พอ
- ใช้สี + ข้อความ ไม่ใช้สีอย่างเดียว
- ถ้าเสียงไม่ดัง ผู้ใช้ยังดูสีและข้อความได้
- ถ้าพื้นที่เสียงดัง ควรให้จอสถานะเปลี่ยนสีชัดเจนมาก

## 13. UX ห้ามทำ

- ห้ามบังคับกด Save หลังสแกน
- ห้ามบังคับกด Enter
- ห้ามให้ผู้ใช้เข้าเมนูหลายชั้น
- ห้ามให้ผู้ใช้ต้องหาไฟล์ CSV เองหลัง Export ถ้ามี Share Button
- ห้ามล้างข้อมูลทันทีโดยไม่ Confirm
